# write-pr-description

Write a pull request description:
- Summary of changes (2-3 sentences covering what and why)
- Motivation: what problem this solves and why this approach
- Testing approach: what was tested, how, and edge cases covered
- Include screenshots or screen recordings for UI changes
- Checklist: reviewed own diff, tests pass, lint clean, docs updated
- Call out breaking changes and link related issues with "Closes #"
