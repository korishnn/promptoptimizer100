# write-rfc

Write a design document (RFC):
- Metadata header: status (draft/proposed/approved), date, authors, PR link
- Summary: one-paragraph executive summary of the proposal
- Motivation: current pain points, why change is needed now
- Design: detailed technical approach, interfaces, data models, diagrams
- Alternatives considered: 2-3 other approaches and why they were rejected
- Tradeoffs: what this design optimizes for and what it sacrifices
- Implementation plan: phased breakdown with milestones
- Open questions: decisions yet to be made
