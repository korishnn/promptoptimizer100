# refactor-simplify

Simplify complex code without changing its behavior:
- Reduce nesting depth with early returns and guard clauses
- Simplify conditionals by extracting boolean expressions into named variables
- Remove unnecessary abstractions, wrapper functions, or indirection layers
- Flatten overly nested class hierarchies into composition where appropriate
- Replace switch statements with lookup maps or polymorphism where clearer
- Prefer composition over inheritance — favor small composed behaviors over deep type trees
