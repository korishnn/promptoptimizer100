# write-commit

Write a git commit message following Conventional Commits:
- Format: `type(scope): description` where type is feat, fix, chore, docs, refactor, test, or perf
- Body explains WHY the change was made, not WHAT changed
- Footer for breaking changes (`BREAKING CHANGE:`) and issue references (`Closes #123`)
- Subject: max 50 characters. Body lines: max 72 characters
- Use imperative mood in the subject line ("add" not "added")
