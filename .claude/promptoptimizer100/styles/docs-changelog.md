# docs-changelog

Generate a changelog for the described changes:
- Follow Keep a Changelog format with reverse chronological order
- Categorize entries: Added, Changed, Deprecated, Removed, Fixed, Security
- Link each entry to the relevant issue or pull request number
- Use semantic versioning with clear version headers and release dates
- Include migration notes for breaking changes within the entry
- Credit external contributors for their pull requests
- Denote unreleased changes under a master heading
