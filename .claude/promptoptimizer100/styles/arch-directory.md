# arch-directory

Design a directory structure for the described project:
- Group by feature (vertical slicing) not by type (horizontal layering)
- Use consistent naming conventions across all directories and files
- Enforce separation of concerns between layers (api, domain, data, ui)
- Define entry point conventions and module exports explicitly
- Establish clear module boundaries with explicit public interfaces
- Keep directory depth shallow — no more than 4 levels deep
- Include test directory structure mirroring source layout
