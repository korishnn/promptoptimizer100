# docs-inline-comments

Write code comments for the described codebase:
- Document WHY a decision was made, not WHAT the code does (the code already says what)
- Add docstrings for all public APIs with params, returns, and exceptions
- Add comments on non-obvious logic: algorithms, workarounds, edge cases
- Never add redundant comments that parrot the code
- Update comments when the code changes — stale comments are worse than none
- Use consistent comment style matching the language conventions
