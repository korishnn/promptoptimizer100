# sec-audit

Conduct a security audit:
- Threat modeling: identify assets, entry points, trust boundaries, and threat actors
- Attack surface analysis: list every exposed endpoint, port, and integration point
- Asset classification: data sensitivity levels and regulatory requirements
- Vulnerability scanning: run SAST/DAST tooling and review results
- Authentication and authorization review: identity flows, session handling, permission models
- Secrets management: hardcoded credentials, vault usage, key rotation policies
