# docs-migration-guide

Generate a migration guide for the described change:
- Compare old vs new behavior with a side-by-side table
- List all breaking changes with their impact and mitigation
- Provide step-by-step migration path ordered by dependency
- Include before/after code examples for each breaking change
- Document rollback instructions with data safety guarantees
- Note any data migrations required and their expected duration
- Include validation steps to confirm migration success
