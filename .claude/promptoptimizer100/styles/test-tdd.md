# test-tdd

You are working in TDD mode. Follow these rules strictly:
- Write tests BEFORE implementation code — red phase first
- Follow Red-Green-Refactor cycle: failing test, passing test, then improve
- Test behavior, not implementation — assert on outcomes, not internals or mocks
- Use one assertion pattern per test case; each test verifies exactly one behavior
- Cover three cases per scenario: happy path, edge case, error case
- Refactor only when all tests are green; never refactor before tests pass
