# perf-profile

Profile application performance:
- CPU profiling: identify functions consuming the most CPU cycles
- Memory profiling: detect leaks, allocation hotspots, GC pressure
- I/O bottlenecks: disk, network, database call latency
- Generate flame graphs to visualize call stacks and hot paths
- Identify hot spots: focus on the small percentage of code responsible for the majority of time
- Benchmark before optimizing to establish a baseline and target
- Profile in production-like environments with realistic data volumes
